Software Engineering Terminology

Term Definition
Software Engineering The process of creating software
Integrated Development Environment (EDI) Combines all of the different tools of software engineering into a single package
Association for Computing Machinery (ACM) Founded in 1947, the ACM is the worlds largest computing society, with a great support network and over 110,000 students. Current enrollment fees for students is $19
Institute of Electronic and Electrical Engineers (IEEE) Founded in 1963, IEEE is an Electronics/Electrical Engineering association that is an industry leader in innovation. Student membership currently costs $10
Software Life Cycle: Developmental Phase Requirement Analysis (figuring out what the software needs to be capable of), Design (planning the creation of the software), Implementation (creating the software), Testing your program
Modular Implementation Breaking the greater code into smaller modules with specific functions
Unified Modeling Language (UML) A visual language that helps programmers write code
Use Case Who Uses the functions of a program
Use Case Diagram Helps visualize the network of people using systems, and how the system will be used
Data Relationships Three kinds; One-to-one (ex. x:x), one-to-many (ex. x:x,y,z), many-to-many (ex. x,y,z:x,y,z)
Design Pattern A predesigned method for solving a common software problem
Software Quality Assurance (SQA) The process of ensuring the software is presented and functional to the highest degree
Software Testing The process of testing the functions of software
Glass-box Testing Testing designed with knowledge of the interworkings of the software
Black-box Testing Testing designed without knowledge of the interworkings of the software
Beta Testing When a preliminary version of the software is given to a select group of users for testing
Alpha Testing When a near-complete version of the software is given to a select group of users for testing
Documentation All of the information that comes with software
User Documentation Explains features of the software and how to use them
System Documentation Explains the interworkigns of the software for maintenece and optimization purposes
Technical Documentation Explains how software should be installed and services
Intellectual Property Law Legal ownership of an idea/good/service. Copyrights, patents, etc.
Software License Legally allows the user ownership of their specific software (not the software as a whole, just the specific copy they purchased)
Software Disclaimer Outlines the limitations of the creator's liability
Best defense against software liability claims the best defense against software liability claims is to apply sound software engineering principles during the software’s development, to use a level of care compatible with the software’s application, and to produce and maintain records that validate these endeavors
